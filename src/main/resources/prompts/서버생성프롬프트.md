# Miracom MES 커스텀 개발 가이드 (Java)

이 문서는 Miracom MES 프로젝트에서 자바 기반의 컨트롤러, 서비스, 모델, 유틸리티, 배치 작업을 개발하기 위한 통합 가이드라인입니다. 모든 개발은 아래 명시된 규칙과 패턴을 엄격히 준수해야 합니다.

---

## 1. 컨트롤러 (Controller) 작성 패턴
모든 API는 인터페이스(Stub/Definition)와 구현체(Implementation)를 분리하여 작성합니다.

### 1.1 인터페이스 (Controller.java)
- **위치**: `kr.co.miracom.custom.[모듈].resource.[complex/simple].[기능].controller`
- **어노테이션**:
    - `@Api(tags = "...")`: Swagger 그룹화 태그 지정
    - `@RequestMapping(path = CustomConst.VER_PATH)`: 공통 API 버전 경로(`/v1`) 지정
    - `@ApiOperation`, `@ApiDoc`: API 기능 설명 및 문서화
- **경로 정의**: `static final String RESOURCE = "모듈/기능명"` 상수를 정의하여 사용합니다.

### 1.2 구현체 (ControllerImpl.java)
- **어노테이션**: `@CrossOrigin`, `@RestController` 필수 적용.
- **구현 방식**: 컨트롤러 내부에 비즈니스 로직을 작성하지 않고, 반드시 **`BeanUtils.get(Service.class).execute(input)`** 패턴으로 서비스를 호출합니다.

---

## 2. 모델 (Model) 작성 패턴
모든 모델 클래스는 `kr.co.miracom.framework.service.Dto`를 상속받아야 합니다.

### 2.1 DB 엔티티 모델 (Entity)
- **위치**: `kr.co.miracom.custom.[모듈].model`
- **어노테이션**: `@DbistTable`, `@DbistPrimaryKey`, `@ResData` 등 사용.
- **필드 구성**: `factoryCode`, `createTime`, `updateTime` 및 `cmf01~30` 필드 포함.

### 2.2 리소스 및 서비스 DTO
- **Info/Lite/All**: 데이터 조회 범위에 따라 상속 구조로 정의.
- **In/Out**: 서비스 요청(`GetListIn` 상속 등) 및 응답을 위한 전용 객체.

### 2.3 연관 관계 매핑
- **`@ResRelation`**: 타 모듈의 정보(제품명, 공정명 등)를 AOP 기반으로 자동 매핑할 때 사용합니다.
    - 예: `@ResRelation(controller = MaterialController.class, field = "matCode", ...)`

---

## 3. 서비스 (Service) 작성 패턴
비즈니스 로직의 핵심은 서비스 계층에서 처리하며, 인터페이스와 구현체로 분리합니다.

### 3.1 인터페이스 (Service.java)
- **위치**: `kr.co.miracom.custom.[모듈].resource.[complex/simple].[기능].service`
- **구현 방식**: `kr.co.miracom.framework.service.Service<In, Out>` 인터페이스를 상속받아 정의합니다.

### 3.2 구현체 (ServiceImpl.java)
- **어노테이션**: `@Service` 필수 적용.
- **구현 방식**:
    - `execute(In input)` 메서드를 오버라이드하여 로직을 구현합니다.
    - DB 처리는 `DbUtils`를 활용하며, 복잡한 로직은 별도의 `private` 메서드나 유틸리티 클래스로 분리합니다.

---

## 4. 유틸리티 클래스 (Utils) 작성 패턴
비즈니스 로직의 공통화나 복잡한 계산 로직은 유틸리티 클래스에 `static` 메서드로 구현합니다.

- **패키지**: `kr.co.miracom.custom.[모듈명].utils`
- **구현 패턴**:
    - 메서드는 `public static`으로 선언.
    - 파라미터 검증 시 `ValueUtils.checkNotEmpty()` 사용.
    - SQL 호출 시 `DbUtils.toSqlPath(Class.class, "file.sql")` 활용.

---

## 4. 배치 작업 (Job) 작성 패턴
스케줄러에 의해 실행되는 작업은 `DbistControllerJob`을 상속받아 구현합니다.

- **패키지**: `kr.co.miracom.custom.[모듈명].job`
- **구현 방식**:
    - `executeJob(Map<K, V> params)` 오버라이드.
    - 실행 인자 추출 -> 서비스 호출 -> 결과를 JSON 형태로 `setResult`에 저장.

---

## 5. 핵심 공통 라이브러리
- **`DbUtils`**: 데이터베이스 CRUD 핵심 도구. (상세 사용법 하단 참조)
- **`ValueUtils`**: 데이터 타입 변환 및 Null 체크 (`isEmpty`, `populate`, `toJsonStr` 등).
- **`BeanUtils`**: Spring Bean 수동 획득 (`BeanUtils.get(Class)`).
- **`AuthUtils`**: 사용자 및 공장 정보 획득.
- **`BizException`**: 규격화된 비즈니스 예외 처리.

---

## 6. DbUtils 상세 사용 케이스

### 6.1 단일 데이터 조회 (Select)
- **PK로 조회**: `Entity entity = DbUtils.select(Entity.class, id);`
- **복합 PK 조회**: `Entity entity = DbUtils.select(Entity.class, DbUtils.toId(key1, key2));`
- **Query로 조회**: `Entity entity = DbUtils.select(Entity.class, query);`
- **Lock 포함 조회**: `Entity entity = DbUtils.selectWithLock(Entity.class, id);` (Select For Update)
- **필수 체크**: `DbUtils.select(Entity.class, query, true);` (데이터가 없으면 `BizException` 발생)
- **단일 값(Scalar) 조회**: `Integer count = DbUtils.select(sqlPath, query, Integer.class);` (Max, Count 등)

### 6.2 다건 및 페이징 조회 (SelectList, SelectPage)
- **목록 조회**: `List<Entity> list = DbUtils.selectList(Entity.class, query);`
- **SQL 파일 기반 목록 조회**:
  ```java
  String sqlPath = DbUtils.toSqlPath(getClass(), "select_custom_data.sql");
  List<ResultDto> list = DbUtils.selectList(sqlPath, inputDto, ResultDto.class);
  ```
- **페이징 조회**:
  ```java
  Page<ResultDto> page = DbUtils.selectPage(sqlPath, inputDto, ResultDto.class);
  ```

### 6.3 데이터 변경 (Insert, Update, Upsert, Delete)
- **단일 처리**:
  - `DbUtils.insert(entity);`
  - `DbUtils.update(entity);`
  - `DbUtils.upsert(entity);` (존재하면 Update, 없으면 Insert)
  - `DbUtils.delete(entity);` (PK 기준 삭제)
- **다건 처리**:
  - `DbUtils.deleteList(Entity.class, query);` (조건에 맞는 모든 데이터 삭제)
- **배치 처리**:
  - `DbUtils.insertBatch(entityList);`
  - `DbUtils.deleteBatch(Entity.class, entityList);` 또는 `DbUtils.deleteBatch(entityList);`

### 6.4 쿼리 보조 기능
- **필터 추가**: `DbUtils.addFilter(query, "fieldName", "=", value);`
- **공장 필터 자동 추가**: `DbUtils.addFactoryFilter(query);` (현재 접속한 `factoryCode` 자동 바인딩)

---

## 7. 기타 준수 사항
- **Lombok**: `@Getter`, `@DtoSetter`, `@ToString(callSuper = true)` 사용.
- **Swagger**: 모든 필드에 `@ApiModelProperty`로 상세 설명 추가.
- **트랜잭션**: `ServiceAspect`에 의해 자동 관리되므로 수동 설정은 지양.
